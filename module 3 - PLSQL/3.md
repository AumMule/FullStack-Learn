# 📄 PL/SQL Revision Sheet (Part 3/4)

## 👁️ 10. Cursors

### Why Cursors?
While `SELECT INTO` can only fetch a single row, a **Cursor** is a database pointer used to retrieve and process **multiple rows** from a query, one row at a time.

---

### A. Explicit Cursor
Declared and controlled manually in code.
#### Syntax
```sql
DECLARE
   CURSOR cursor_name IS query;
BEGIN
   OPEN cursor_name;
   FETCH cursor_name INTO variables;
   CLOSE cursor_name;
END;
```
#### Example
```sql
DECLARE
   CURSOR c IS SELECT first_name FROM employees WHERE department_id = 20;
   v_name employees.first_name%TYPE;
BEGIN
   OPEN c;
   LOOP
      FETCH c INTO v_name;
      EXIT WHEN c%NOTFOUND;
      DBMS_OUTPUT.PUT_LINE('Employee: ' || v_name);
   END LOOP;
   CLOSE c;
END;
/
```

---

### B. Cursor FOR Loop (Most Common)
Implicitly opens, fetches, and closes the cursor automatically. This is clean, safe, and prevents memory leaks.
#### Syntax
```sql
FOR record_name IN cursor_name LOOP
   -- Access record_name.column
END LOOP;
```
#### Example
```sql
BEGIN
   FOR emp IN (SELECT first_name, salary FROM employees WHERE department_id = 20) LOOP
      DBMS_OUTPUT.PUT_LINE(emp.first_name || ' earns ' || emp.salary);
   END LOOP;
END;
/
```

## 🏗️ 11. Procedures

### Syntax
```sql
CREATE [OR REPLACE] PROCEDURE procedure_name [(parameters)]
IS
BEGIN
   -- Executable logic (No return statement required)
END;
/
```

### Use Case
Equivalent to a `void` method in Java. Used to group a set of SQL statements and procedural commands to execute an operation.

### Example
```sql
CREATE OR REPLACE PROCEDURE greet
IS
BEGIN
   DBMS_OUTPUT.PUT_LINE('Hello, PL/SQL User!');
END;
/
```

## 📐 12. Functions

### Syntax
```sql
CREATE [OR REPLACE] FUNCTION function_name [(parameters)]
RETURN return_datatype
IS
BEGIN
   -- Logic
   RETURN return_value;
END;
/
```

### Use Case
Equivalent to value-returning methods in other languages. Must return a single value of a specified data type.

### Example
```sql
CREATE OR REPLACE FUNCTION add_nums(a NUMBER, b NUMBER)
RETURN NUMBER
IS
BEGIN
   RETURN a + b;
END;
/
```

## 🎛️ 13. Parameters

PL/SQL parameters control the flow of data in and out of stored procedures or functions.

| Mode | Flow | Behavior |
| :--- | :--- | :--- |
| **`IN`** | Input only (Default) | Read-only inside the procedure. |
| **`OUT`** | Output only | Writes values back to the calling block. Uninitialized on start. |
| **`IN OUT`** | Input + Output | Read value on start, write value back on exit. |

### Syntax & Example
```sql
CREATE OR REPLACE PROCEDURE modify_salary(
   p_emp_id IN NUMBER,               -- Input parameter
   p_bonus  IN NUMBER,               -- Input parameter
   p_new_sal OUT NUMBER,             -- Output parameter
   p_io_flag IN OUT VARCHAR2         -- Input & Output parameter
)
IS
BEGIN
   -- Read input and write to output
   SELECT salary + p_bonus INTO p_new_sal 
   FROM employees 
   WHERE employee_id = p_emp_id;

   -- Modify the IN OUT parameter
   p_io_flag := 'PROCESSED: ' || p_io_flag;
END;
/
```
