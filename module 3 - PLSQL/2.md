## 🚦 6. IF / ELSIF / ELSE

### Syntax
```sql
IF condition1 THEN
   -- statements
ELSIF condition2 THEN
   -- statements
ELSE
   -- statements
END IF;
```

### Use Case
Allows branching control flows depending on whether conditions evaluate to `TRUE`, `FALSE`, or `NULL`.

### Example
```sql
DECLARE
   marks NUMBER := 85;
BEGIN
   IF marks >= 90 THEN
      DBMS_OUTPUT.PUT_LINE('Grade: A');
   ELSIF marks >= 70 THEN
      DBMS_OUTPUT.PUT_LINE('Grade: B');
   ELSE
      DBMS_OUTPUT.PUT_LINE('Grade: C');
   END IF;
END;
/
```

## 🎛️ 7. CASE

### Syntax
```sql
CASE selector
   WHEN value1 THEN -- statements
   WHEN value2 THEN -- statements
   ELSE -- default statements
END CASE;
```

### Use Case
Evaluates multiple values in a concise format. Can be written as a simple CASE (matching values of a selector) or a searched CASE (checking boolean conditions).

### Example
```sql
DECLARE
   grade CHAR(1) := 'B';
BEGIN
   CASE grade
      WHEN 'A' THEN DBMS_OUTPUT.PUT_LINE('Excellent');
      WHEN 'B' THEN DBMS_OUTPUT.PUT_LINE('Very Good');
      ELSE DBMS_OUTPUT.PUT_LINE('Needs Improvement');
   END CASE;
END;
/
```

## 🔄 8. Loops

PL/SQL provides three main loop constructs.

### A. Basic LOOP (Infinite with Exit)
Iterates indefinitely until an explicit exit statement is encountered.
#### Syntax
```sql
LOOP
   -- statements
   EXIT WHEN condition;
END LOOP;
```
#### Example
```sql
DECLARE
   i NUMBER := 1;
BEGIN
   LOOP
      DBMS_OUTPUT.PUT_LINE('Iteration: ' || i);
      i := i + 1;
      EXIT WHEN i > 3;
   END LOOP;
END;
/
```

### B. WHILE Loop
Executes statements as long as the condition evaluates to `TRUE`.
#### Syntax
```sql
WHILE condition LOOP
   -- statements
END LOOP;
```
#### Example
```sql
DECLARE
   i NUMBER := 1;
BEGIN
   WHILE i <= 3 LOOP
      DBMS_OUTPUT.PUT_LINE('Iteration: ' || i);
      i := i + 1;
   END LOOP;
END;
/
```

### C. FOR Loop
Iterates through a fixed number of index values.
#### Syntax
```sql
FOR index IN [REVERSE] lower_bound..upper_bound LOOP
   -- statements
END LOOP;
```
#### Example
```sql
BEGIN
   FOR i IN 1..3 LOOP
      DBMS_OUTPUT.PUT_LINE('Iteration: ' || i);
   END LOOP;
END;
/
```

## ⚠️ 9. Exceptions

### 🟥 A. Predefined Exceptions
Built-in database runtime errors that are raised automatically.
*   **`NO_DATA_FOUND`**: Raised when a `SELECT INTO` returns 0 rows.
*   **`TOO_MANY_ROWS`**: Raised when a `SELECT INTO` returns more than 1 row.
*   **`ZERO_DIVIDE`**: Raised when division by zero is attempted.

### 🟨 B. Generic Exceptions
Catches any database error not caught by previous handlers using the `OTHERS` wildcard keyword.
```sql
EXCEPTION
   WHEN OTHERS THEN
      DBMS_OUTPUT.PUT_LINE('An unexpected error occurred.');
```

### 🟦 C. User-Defined Exceptions
Errors declared and raised explicitly by code when business rules are violated.
#### Syntax
```sql
DECLARE
   exception_name EXCEPTION;
BEGIN
   ...
   RAISE exception_name;
```
#### Example
```sql
DECLARE
   my_error EXCEPTION;
   v_age NUMBER := 15;
BEGIN
   IF v_age < 18 THEN
      RAISE my_error;
   END IF;
EXCEPTION
   WHEN my_error THEN
      DBMS_OUTPUT.PUT_LINE('Custom Error: Customer is underage.');
   WHEN OTHERS THEN
      DBMS_OUTPUT.PUT_LINE('Unexpected error occurred.');
END;
/
```
