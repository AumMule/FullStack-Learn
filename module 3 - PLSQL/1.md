# 📄 PL/SQL Revision Sheet (Part 1/4)

## 🧱 1. PL/SQL Block Structure

### Syntax
```sql
[DECLARE]
   -- Variables, constants, cursors declaration (Optional)
BEGIN
   -- Executable statements / Logic (Mandatory)
[EXCEPTION]
   -- Error handling rules (Optional)
END;
/
```

### Use Case
The basic wrapper for any PL/SQL program. Used to group statements together for execution in the database engine.

### Example
```sql
BEGIN
   DBMS_OUTPUT.PUT_LINE('PL/SQL Block Executed Successfully.');
END;
/
```

---

## 📝 2. Variables

### Syntax
```sql
DECLARE
   variable_name datatype [:= initial_value];
```

### Use Case
Used to store data values temporarily during the execution of a PL/SQL block.

### Example
```sql
DECLARE
   sal NUMBER;
   name VARCHAR2(50);
BEGIN
   sal := 50000;
   name := 'Aum Patel';
   DBMS_OUTPUT.PUT_LINE(name || ' earns ' || sal);
END;
/
```

## 🔗 3. `%TYPE`

### Syntax
```sql
variable_name table_name.column_name%TYPE;
```

### Use Case
Copies the exact data type and scale of a database column. If the table's column changes size or type, your PL/SQL code adapts automatically.

### Example
```sql
DECLARE
   -- Copies the exact datatype of the salary column in employees table
   v_salary employees.salary%TYPE; 
BEGIN
   SELECT salary INTO v_salary FROM employees WHERE employee_id = 101;
   DBMS_OUTPUT.PUT_LINE('Salary is: ' || v_salary);
END;
/
```

## 📑 4. `%ROWTYPE`

### Syntax
```sql
record_name table_name%ROWTYPE;
```

### Use Case
Creates a record variable that matches the entire row structure of a database table. Fields within the record are accessed using dot notation (`record.field`).

### Example
```sql
DECLARE
   -- Creates a record structure containing all columns of employees table
   emp employees%ROWTYPE; 
BEGIN
   SELECT * INTO emp FROM employees WHERE employee_id = 101;
   
   -- Accessing fields within the record
   DBMS_OUTPUT.PUT_LINE('Employee: ' || emp.first_name || ' ' || emp.last_name);
   DBMS_OUTPUT.PUT_LINE('Salary  : ' || emp.salary);
END;
/
```

## 📥 5. `SELECT INTO`

### Syntax
```sql
SELECT column1, column2 
INTO variable1, variable2 
FROM table_name 
WHERE condition;
```

### Use Case
Retrieves data from a table and assigns it to local variables. 
> [!IMPORTANT]
> The query **MUST** return exactly one row.

### Core Exceptions
*   **`NO_DATA_FOUND`**: Thrown if the query returns 0 rows.
*   **`TOO_MANY_ROWS`**: Thrown if the query returns 2 or more rows.

### Example
```sql
DECLARE
   v_name employees.first_name%TYPE;
BEGIN
   SELECT first_name INTO v_name FROM employees WHERE employee_id = 1;
   DBMS_OUTPUT.PUT_LINE('Name: ' || v_name);
EXCEPTION
   WHEN NO_DATA_FOUND THEN
      DBMS_OUTPUT.PUT_LINE('Error: No employee found with that ID.');
   WHEN TOO_MANY_ROWS THEN
      DBMS_OUTPUT.PUT_LINE('Error: Multiple employees found.');
END;
/
```
